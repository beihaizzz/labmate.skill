# Lab Report JSON Schemas

This document defines the JSON data structures used by the LabMate skill.

---

## progress.json Schema

Tracks the experiment progress state.

```json
{
  "experiment_name": "string",
  "total_steps": "int",
  "current_step": "int",
  "completed_steps": ["int"],
  "screenshots_required": [
    {
      "step": "int",
      "description": "string",
      "captured": "bool",
      "path": "string|null"
    }
  ],
  "notes": {"step_N": "string"},
  "last_updated": "ISO8601 datetime",
  "status": "not_started | in_progress | completed"
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `experiment_name` | string | Name of the experiment |
| `total_steps` | int | Total number of steps in the experiment |
| `current_step` | int | Current step being worked on (1-indexed) |
| `completed_steps` | [int] | List of completed step numbers |
| `screenshots_required` | array | List of screenshot requirements per step |
| `notes` | object | Step-level notes, key format: "step_N" |
| `last_updated` | string | ISO8601 datetime of last update |
| `status` | string | Overall progress status: not_started, in_progress, or completed |

## student-info.json Schema

Stores student identification information.

```json
{
  "姓名": "string",
  "学号": "string",
  "学院": "string",
  "专业": "string",
  "班级": "string"
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `姓名` | string | Student's full name |
| `学号` | string | Student ID number |
| `学院` | string | College/faculty name |
| `专业` | string | Major/discipline name |
| `班级` | string | Class/turor group |

## feedback_report.json Schema

### Purpose
Structured feedback report generated by `/lab -review` for skill self-optimization.

### Schema
```json
{
  "skill_version": "string — current LabMate version (e.g., v1.0.0)",
  "project": "string — project path",
  "generated_at": "ISO8601 timestamp",
  "task_completion": {
    "guide_mode": {"total_steps": "int", "completed_autonomously": "int", "required_intervention": "int", "failed": "int"},
    "work_mode": {"template_filled": "bool", "manual_cells_needed": "bool", "images_inserted": "int"},
    "attribution": [{"module": "template_filling|guide_parsing|compatibility|image_handling|git_management", "success_count": "int", "failure_count": "int", "notes": "string"}]
  },
"issues": [{
    "id": "string",
    "severity": "critical|high|medium|low",
    "title": "string",
    "module": "string",
    "description": "string",
    "repro_steps": "string",
    "affected_files": ["string"],
    "commit_reference": "string|null"
  }],
  "issue_templates": [{
    "title": "[skill optimization] <module>: <summary>",
    "labels": ["skill-optimization", "<module>"],
    "body": "## Impact\n\n## Reproduction\n\n## Current Behavior\n\n## Expected Behavior\n\n## Improvement Suggestion\n\n## Related Commits"
  }]
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `skill_version` | string | Current LabMate version |
| `project` | string | Project root path |
| `generated_at` | string | ISO8601 timestamp of report generation |
| `task_completion` | object | Task completion metrics and module attribution |
| `task_completion.guide_mode` | object | Guide Mode step completion stats |
| `task_completion.work_mode` | object | Work Mode template filling stats |
| `task_completion.attribution` | array | Per-module success/failure attribution |
| `issues` | array | List of identified issues with severity and module |
| `issue_templates` | array | Pre-formatted GitHub Issue templates |
